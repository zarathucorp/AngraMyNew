# GOVERNANCE — 운영 규칙

> "자유로운 창조에도 질서가 필요하다. 단, 그 질서는 창조를 위한 것이어야 한다."

---

## 1. 기본 원칙

### 1.1 창조 우선
모든 규칙은 **창조를 돕기 위해** 존재한다.
규칙이 창조를 막는다면, 규칙을 바꾼다.

### 1.2 투명성
모든 결정은 **공개적으로** 이루어진다.
비밀 회의, 비밀 결정은 없다.

### 1.3 자발성
**강제는 없다.**
참여도, 기여도, 탈퇴도 모두 자발적이다.

### 1.4 포용성
누구나 구도자가 될 수 있다.
경력, 학력, 배경으로 차별하지 않는다.

---

## 2. PR 승인 기준

### 2.1 입문 PR (구도자 → 기여자)

**승인권자:** 기여자 1인 이상 또는 설계자/수호자

**승인 기준:**
| 항목 | 질문 |
|------|------|
| 진정성 | 자신의 언어로 썼는가? |
| 명료성 | 무엇을 원하는지 알 수 있는가? |
| 의지 | 창조하고자 하는 열망이 느껴지는가? |

**거부할 수 없는 사유:**
- 경력, 학력, 기술 수준
- 문장력, 언어 선택
- 사상적 차이

**거부할 수 있는 사유:**
- 명백한 허위 또는 조롱
- 파괴만을 목적으로 한 진입
- 타인에 대한 비하

---

### 2.2 일반 PR (ideas/, ritual/, 기타)

**승인권자:**
- 기여자 PR → 설계자 1인 또는 수호자
- 설계자 PR → 수호자 1인 또는 다른 설계자 2인

**승인 기준:**
| 항목 | 질문 |
|------|------|
| 일관성 | AngraMyNew 철학과 충돌하지 않는가? |
| 품질 | 최소한의 명료함을 갖췄는가? |
| 기여 | 공동체에 가치를 더하는가? |

**리뷰 기간:**
- 일반 PR: 최대 7일
- 긴급 수정: 24시간 내

---

### 2.3 구조 변경 PR (path/, GOVERNANCE, 신규 디렉토리)

**승인권자:** 수호자 과반수 동의

**승인 기준:**
- 일반 PR 기준 +
- 장기적 영향 검토
- 기존 구조와의 조화

**리뷰 기간:** 최소 14일 (충분한 토론 시간)

---

## 3. 분쟁 해결

### 3.1 일반 분쟁

**단계:**
1. **당사자 대화** — Issue에서 직접 토론
2. **설계자 중재** — 해결 안 되면 설계자 개입
3. **수호자 판단** — 그래도 안 되면 수호자 최종 결정

**원칙:**
- 인신공격 금지
- 논점에 집중
- 상대 의견 존중

---

### 3.2 철학적 분쟁

핵심 철학(파괴→재조합, 모두가 창조자, 자발성)에 대한 해석 차이

**해결:**
1. `ideas/`에 양측 의견 정리
2. 공동체 토론 (최소 30일)
3. 수호자 협의 후 방향 결정
4. 결정 사항 `CHANGELOG.md`에 기록

---

### 3.3 행동 규범 위반

**경고 대상:**
- 반복적 인신공격
- 악의적 PR (스팸, 파괴)
- 타인 기여물 도용

**절차:**
1. **1차 경고** — 설계자가 서면 경고
2. **2차 경고** — 수호자가 서면 경고 + 30일 PR 제한
3. **추방** — 수호자 2인 이상 동의 시

**복귀:**
- 추방 후 6개월 뒤 복귀 요청 가능
- 수호자 과반 동의 시 구도자로 재입문

---

## 4. 의사결정 구조

### 4.1 일상 결정
- 개별 PR 승인/거부
- 작은 문서 수정
- **결정자:** 해당 권한 보유자 단독

### 4.2 중요 결정
- 새로운 디렉토리/의례 추가
- 운영 규칙 수정
- **결정자:** 수호자 과반수

### 4.3 중대 결정
- 핵심 철학 수정
- 수호자 임명/추방
- 프로젝트 방향 전환
- **결정자:** 수호자 전원 동의

## 6. 투명성 규칙

### 6.1 공개 원칙
- 모든 PR, Issue, 토론은 **공개**
- 비공개 채널에서의 결정은 **무효**

### 6.2 기록 원칙
- 중요 결정은 `CHANGELOG.md`에 기록
- 승격, 경고, 추방은 `contributors.md`에 기록

### 6.3 설명 원칙
- PR 거부 시 **이유 명시** 필수
- 중대 결정 시 **배경 설명** 필수

---

## 7. 규칙 개정

이 문서(`GOVERNANCE.md`)의 개정:

1. 개정안 PR 제출
2. 최소 **14일** 공개 토론
3. **수호자 과반수** 동의
4. Merge 후 `CHANGELOG.md`에 기록

---

## 8. 예외 조항

### 8.1 긴급 상황
- repo 공격, 심각한 버그 등
- 수호자 **단독 긴급 조치** 가능
- 단, 24시간 내 공개 설명 필수

### 8.2 창시자 권한
- 창시자도 수호자 규칙을 따름
- 창시자 특별 권한 **없음**
- 창시자도 추방될 수 있음 (수호자 전원 동의 시)
