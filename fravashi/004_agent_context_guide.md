# AngraMyNew — Project Context for Claude

## 프로젝트 정체성

AngraMyNew는 **철학·예술·실험 운동**이다. Quarto Book으로 빌드되며 HTML/PDF/EPUB/DOCX 출력.
- **AngraMyNew = Angra Mainyu + My + New** — 파괴의 신에서 나만의 새 세계로
- AngraMyNew는 **장치(Device)**이지 정체성이 아님. "나는 AngraMyNew다" 같은 선언 금지 (v6.0에서 제거됨)
- 교리를 제공하지 않음. 실험 장치(정신의 LHC)

## 3대 공리

1. **파괴의 공리**: 자기정화. 칼날은 바깥을 향하지 않는다
2. **창조의 공리**: 절대적 아름다움. 꽃은 벌과 논쟁하지 않는다
3. **확장의 공리**: 데뷔. 나의 'My' 완성 후 타인의 'My'를 데뷔시킨다

## 핵심 용어

| 용어 | 정의 | 주의사항 |
|------|------|----------|
| **면세인** | 시스템의 과세권에서 탈거한 자 | 가난한 자가 아님 |
| **징세인** | 세계관의 밀도로 가치를 끌어당기는 자 | 면세를 거치지 않으면 자격 없음 |
| **악상(惡想)** | 정돈 이전의 진동. 논리보다 먼저 온 감각 | Musical Idea가 아님. 022에서 정의 |
| **진선미 좌표계** | 진(테크노봉건) / 선(관료주의) / 미(미적군벌연합) | 026 삼국지 비유 |
| **밀도/곡률/중력** | 세계관의 무게가 주변을 휘게 만드는 것 | 뉴턴(힘) 아님, 아인슈타인(곡률) |
| **섭취/상환** | 생존은 섭취의 결과, 창조는 상환 | 빚/대속/신성한 등 종교어 사용 금지 |

## 세계관 규칙 (수정 이력 기반)

- **"우리" 최소화**: "우리는~" 대신 주어 생략 또는 구체적 주어 사용
- **종교어 금지**: 빚, 대속, 신성한, 우주적 횡령, 흡혈귀 → 섭취, 상환, 밀도, 미상환으로 대체
- **천(天) 금지, 천하(天下) 허용**: 천=종교적 권위(금지), 천하=나의 영토(허용)
- **"아름다움은 영원하다" 금지**: 파괴의 공리, "정체는 연체다"와 모순
- **숭례문 등 문화재 폄하 금지**: 그 자체가 문화적 중력
- **AngraMyNew = 장치**: "나는 AngraMyNew다" 같은 정체성 선언 금지
- **교리/사상 제공 안 함**: "사상을 제공하지 않는다" → "교리를 제공하지 않는다"로 수정 완료
- **자기검열 금지**: 024에서 "재구성된 형식만 사용" 제거. 극단적 사고실험 허용

## 글쓰기 스타일 가이드라인

### 장르 구분

- **art/** = 이해시키는 문서. "이것이 왜 아름다운가"를 독자가 납득하게 만든다
- **ideas/** = 기준을 정하고 행동을 바꾸는 문서. 읽고 나면 판단 기준이 달라져야 한다

### 최상위 원칙

**독자의 논리와 의식의 흐름대로 읽혀야 한다.** 문장이 예쁜 것보다 생각이 순서대로 쌓이는 것이 중요하다. "이런 문제가 있다 → 왜 그런가 → 예를 들면 → 그러면 어떻게 하나 → 이렇게 하면 된다." 독자가 다음에 궁금해할 것을 그 타이밍에 던진다. 수사 순서(멋있는 문장 나열)가 아니라 논리 순서로 쓴다.

### 가독성 원칙 (최우선 — 다른 모든 스타일 규칙보다 위)

- **쉬운 말로 써라**: 한자어 연타 금지. "욕망과 주의와 접근성" → "사람이 몰리고, 눈길이 쏠리고". 읽다 멈추게 만드는 단어는 틀린 단어다
- **같은 말 두 번 하지 마라**: 한 번 말했으면 끝. 표현만 바꿔서 반복하는 것도 반복이다
- **설명하고 또 보여주지 마라**: 개념을 설명한 다음 예시로 또 보여주면 두 번이다. 보여주는 것으로 충분하면 설명을 빼라
- **한 섹션 한 주어**: 포주 얘기하다 갑자기 다른 주체가 나오면 안 된다. 주제에서 이탈하지 마라
- **어설픈 비유는 직접 말하는 것보다 못하다**: "칼날을 쥘 수 없는 구조" → "끊을 수 없는 구조". 비유 없이 말할 수 있으면 비유를 쓰지 마라
- **안 맞는 비교는 빼라**: 현실에서 안 맞는 비교는 억지다. 안 쓰는 게 낫다
- **원인을 복잡하게 포장하지 마라**: 단순한 건 단순하게. "벌어도 남지 않는 구조에서" → "몸이 닳으니까"
- **줄일 수 있으면 줄여라**: 4문장이 1문장이 될 수 있다. 앞에서 다 말했으면 맺음은 한 방이면 된다

### 공통 규칙 (art/ + ideas/ 모두 적용)

- **자연스러운 복문 사용**: ~는데, ~이며, ~므로, ~니까 등으로 문장을 이어간다. 단문 남발 금지
- **볼드 절제**: 핵심 용어 첫 등장에만. 강조 남발 금지
- **추상적 주장마다 구체적 앵커 1개**: 숫자, 수식, 사례, 역사적 사실 중 하나. 근거 없는 추상어 반복 금지
- **반론/한계 최소 1곳 포함**: "얼핏 보면 ~같은 느낌이 드는데, 전혀 아니다" 같은 전환이 구조적으로 존재해야 한다
- **수사 패턴 반복 금지**: "X가 아니다. Y인 것이다" 같은 공식 반복 금지
- **독자 대화체 허용**: ~해보자, ~라고 생각하면 된다, ~인 셈이다
- **"~라고 생각한다" 헤징 금지**: 직접 서술. "이 대목이라고 생각한다" → "이 대목이다"
- **선언문 톤 금지**: 흐르다가 가끔 치는 리듬. 모든 문장이 선언이면 AI가 쓴 것처럼 보인다
- **구어 펀치 허용**: "시발비용", "쪽수가 깡패" 같은 현실 언어가 학술 용어보다 세다
- **질문으로 전환**: 수사적 질문이 아니라 다음 단락에서 진짜 답하는 질문
- **맺음은 짧게 닫는다**: 앞에서 다 말했으면 1-2줄이면 된다. 요약하지 마라

### art/ 전용

- 구조: 맥락/배경 → 아름다운 것 → 왜 아름다운가 → 다른 art/ 사례와 연결 → 맺음
- 목표: 독자가 "아, 그래서 이게 아름다운 거구나"라고 납득하는 것
