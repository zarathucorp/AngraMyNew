# 악상 스튜디오 — AI 이전 단계의 창작 도구

> 관련: ideas/022(악상의 시대), ideas/017(증명의 아름다움), fravashi/001(패턴 추출 5단계)

---

## 개요

모든 AI 도구가 "잘 정리된 프롬프트"를 전제한다.
악상 스튜디오는 프롬프트 이전 단계 — "뭘 만들지 모르는 상태"를 다루는 유일한 도구.

---

## 구조

### 입력 (비구조적)
- 음성 중얼거림
- 낙서/스케치
- 키워드 나열
- 불쾌감/끌림 메모
- 스크린샷/사진 모음

### 처리 (정리하지 않는다)
AI가 "정리"하지 않고 **패턴만 시각화**:
- 반복어 하이라이트
- 연결선 (관련 키워드 클러스터링)
- 밀도맵 (어디에 에너지가 몰리는지)
- 시간축 (어떤 악상이 반복 출현하는지)

### Fravashi 엔진 내장 — 패턴 추출 5단계 UI
| 단계 | 이름 | 설명 |
|------|------|------|
| 0 | 악상 | 정돈 이전의 진동 — 그대로 기록 |
| 1 | 욕망 | 무엇에 끌리는가 |
| 2 | 반복 | 무엇이 계속 나타나는가 |
| 3 | 충돌 | 무엇이 부딪히는가 |
| 4 | 방향성 | 에너지가 어디로 흐르는가 |
| 5 | 원형 | 상징적 구조 |

### 출력
- 유저가 스스로 구조를 발견하면 → 기존 AI 도구(GPT/Claude/Midjourney)로 핸드오프
- "악상 → 프롬프트" 변환 브릿지

---

## 왜 되는가

- 창작의 진짜 병목은 "뭘 만들지 모르는 상태" — 이 구간을 다루는 도구가 없음
- 022의 핵심: "악상은 정보가 아니다. 정돈되기 이전의 진동이다"
- 017의 핵심: 인지적 엔트로피를 낮추는 게 아니라, 높은 상태를 **견딜 수 있게** 해주는 도구
- 기존 AI 도구와 경쟁이 아니라 **전 단계**: 악상 스튜디오에서 시작 → GPT/Claude로 마무리

---

## 타깃 유저

- 작가/아티스트: "뭔가 만들고 싶은데 뭔지 모르겠다"
- 기획자/PD: "콘셉트가 안 잡힌다"
- 창업자: "문제는 느끼는데 정의가 안 된다"
- 연구자: "가설 이전의 직감을 정리하고 싶다"

---

## 수익 모델

- 구독 SaaS (개인 크리에이터)
- 팀 플랜 (디자인 스튜디오, 에이전시, R&D팀)
- AI 도구 연동 수수료 (프롬프트 핸드오프 시)

---

## MVP 로드맵

1. 웹 앱: 텍스트/이미지 자유 입력 → 키워드 클러스터 시각화
2. 반복 패턴 탐지 추가
3. Fravashi 5단계 UI 구현
4. GPT/Claude API 연동 (악상 → 프롬프트 변환)
5. 음성 입력 추가
